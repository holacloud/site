# 配置

Config 是 Hola.Cloud 生态系统中的集中式配置管理服务。它能够以分层、版本化和安全的方式存储、检索和管理应用程序配置。

## 关键优势

### 分层配置
Config 支持项目、环境和服务的分层结构。这允许您定义基础配置，并根据环境（开发、预发布、生产）或服务覆盖特定值，从而减少重复并简化管理。

### 基于 JSON
所有配置都存储为 JSON 文档，易于读写，并能与任何编程语言或工具链集成。模式灵活，您可以存储任何有效的 JSON 结构。

### 版本化管理
每次配置更新都会创建新版本。您可以跟踪变更历史、比较修订版本，并在需要时回滚到之前的版本。

### 易于集成
Config 可与 Hola.Cloud 生态系统中的其他服务（如 InceptionDB、Lambda 和 InstantLogs）无缝集成。在服务之间集中配置可减少重复工作，并保持基础设施的一致性。

### 安全访问
该服务提供两个 API 接口。管理 API（`/v0/configs`）对管理操作公开访问。用户 API（`/v1/config`）需要 API 密钥，确保只有授权客户端才能读取或修改运行时配置。

## API 概览

| 方法 | 路径 | 描述 | 认证 |
|--------|------|-------------|------|
| GET | `/v0/configs` | 列出所有配置（管理） | 公开 |
| POST | `/v0/configs` | 创建新配置（管理） | 公开 |
| GET | `/v0/configs/{id}` | 根据 ID 获取配置（管理） | 公开 |
| DELETE | `/v0/configs/{id}` | 删除配置（管理） | 公开 |
| PATCH | `/v0/configs/{id}` | 部分更新配置（管理） | 公开 |
| GET | `/v1/config` | 获取当前用户配置 | API 密钥 |
| PATCH | `/v1/config` | 更新当前用户配置 | API 密钥 |

基础 URL：`https://api.hola.cloud`

## 最佳使用场景

### 微服务配置
集中管理并版本控制所有微服务的配置。每个服务在启动时获取自己的配置，并在无需重新部署的情况下接收更新。

### 多环境部署
从单一控制面板管理开发、预发布和生产环境的独立配置。仅覆盖不同环境之间的差异值。

### 功能开关
将功能切换标志存储为配置项，并在运行时更新而无需修改代码。PATCH 端点支持部分更新，使单个标志的切换更加高效。
