Trabajando con Archivos

Esta guía cubre las operaciones con archivos en HolaCloud Files: subir, descargar, listar, obtener metadatos y eliminar archivos.

Rutas de Archivos

Las rutas de los archivos se especifican en la URL después de 

1/files/
o 
1/list/
. Las rutas pueden incluir directorios usando barras inclinadas:

1/v1/buckets/{id}/files/images/logo.png
2/v1/buckets/{id}/files/backups/db-2026-06-21.sql.gz

No es necesario crear directorios previamente -- la ruta del archivo se trata como una clave plana con barras.

Subir un Archivo

Use PUT para subir un archivo. Establezca el encabezado 

1Content-Type
al tipo MIME apropiado.

Subir un Archivo de Texto

1curl -X PUT "https://api.hola.cloud/v1/buckets/bkt_abc123/files/notas/leame.txt" \
2  -H "Api-Key: SU_API_KEY" \
3  -H "Api-Secret: SU_API_SECRET" \
4  -H "Content-Type: text/plain" \
5  --data-binary @leame.txt

Subir un Archivo Binario (Imagen)

1curl -X PUT "https://api.hola.cloud/v1/buckets/bkt_abc123/files/images/foto.jpg" \
2  -H "Api-Key: SU_API_KEY" \
3  -H "Api-Secret: SU_API_SECRET" \
4  -H "Content-Type: image/jpeg" \
5  --data-binary @foto.jpg

Subir con Metadatos

Puede adjuntar metadatos personalizados a los archivos usando el encabezado 

1File-Metadata
:

1curl -X PUT "https://api.hola.cloud/v1/buckets/bkt_abc123/files/informe.pdf" \
2  -H "Api-Key: SU_API_KEY" \
3  -H "Api-Secret: SU_API_SECRET" \
4  -H "Content-Type: application/pdf" \
5  -H "File-Metadata: {\"author\": \"Ana García\", \"department\": \"ingeniería\"}" \
6  --data-binary @informe.pdf

Respuesta esperada:

 1{
 2  "path": "informe.pdf",
 3  "size": 245760,
 4  "contentType": "application/pdf",
 5  "uploadedAt": "2026-06-21T12:00:00Z",
 6  "etag": "\"abc123def456\"",
 7  "metadata": {
 8    "author": "Ana García",
 9    "department": "ingeniería"
10  }
11}

Descargar un Archivo

Use GET para descargar un archivo por su ruta:

1curl "https://api.hola.cloud/v1/buckets/bkt_abc123/files/images/foto.jpg" \
2  -H "Api-Key: SU_API_KEY" \
3  -H "Api-Secret: SU_API_SECRET" \
4  -o foto.jpg

Solicitud HTTP:

1GET /v1/buckets/bkt_abc123/files/images/foto.jpg HTTP/1.1
2Host: api.hola.cloud
3Api-Key: SU_API_KEY
4Api-Secret: SU_API_SECRET

La respuesta incluye el contenido del archivo con los encabezados 

1Content-Type
y 
1Content-Length
 apropiados.

Descargas Parciales (Solicitudes de Rango)

Files soporta solicitudes de rango HTTP para descargas parciales:

1curl "https://api.hola.cloud/v1/buckets/bkt_abc123/files/video.mp4" \
2  -H "Api-Key: SU_API_KEY" \
3  -H "Api-Secret: SU_API_SECRET" \
4  -H "Range: bytes=0-1023" \
5  -o fragmento_video.mp4

Listar Archivos

Liste archivos en un bucket con un filtro de prefijo opcional. El 

1*
después de 
1/list/
 es obligatorio e indica todos los archivos. Use una ruta específica para filtrar por prefijo.

Listar Todos los Archivos

1curl "https://api.hola.cloud/v1/buckets/bkt_abc123/list/*" \
2  -H "Api-Key: SU_API_KEY" \
3  -H "Api-Secret: SU_API_SECRET"

Listar Archivos en un Directorio

Liste archivos bajo el prefijo 

1images/
:

1curl "https://api.hola.cloud/v1/buckets/bkt_abc123/list/images/*" \
2  -H "Api-Key: SU_API_KEY" \
3  -H "Api-Secret: SU_API_SECRET"

Respuesta:

 1{
 2  "files": [
 3    {
 4      "path": "images/foto.jpg",
 5      "size": 245760,
 6      "contentType": "image/jpeg",
 7      "modifiedAt": "2026-06-21T12:00:00Z"
 8    },
 9    {
10      "path": "images/banner.png",
11      "size": 512000,
12      "contentType": "image/png",
13      "modifiedAt": "2026-06-21T11:30:00Z"
14    }
15  ],
16  "prefix": "images/",
17  "total": 2
18}

Listar con Paginación

1curl "https://api.hola.cloud/v1/buckets/bkt_abc123/list/*?limit=10&offset=0" \
2  -H "Api-Key: SU_API_KEY" \
3  -H "Api-Secret: SU_API_SECRET"

Obtener Metadatos de un Archivo

Use HEAD para obtener metadatos de un archivo sin descargar el contenido:

1curl -I "https://api.hola.cloud/v1/buckets/bkt_abc123/files/informe.pdf" \
2  -H "Api-Key: SU_API_KEY" \
3  -H "Api-Secret: SU_API_SECRET"

Encabezados de respuesta:

1HTTP/1.1 200 OK
2Content-Type: application/pdf
3Content-Length: 245760
4ETag: "abc123def456"
5Last-Modified: Sun, 21 Jun 2026 12:00:00 GMT
6File-Metadata: {"author": "Ana García", "department": "ingeniería"}

Eliminar un Archivo

Elimine un archivo por su ruta:

1curl -X DELETE "https://api.hola.cloud/v1/buckets/bkt_abc123/files/notas/leame.txt" \
2  -H "Api-Key: SU_API_KEY" \
3  -H "Api-Secret: SU_API_SECRET"

Respuesta esperada: HTTP 204 Sin Contenido.

Error: Archivo No Encontrado

1{
2  "error": {
3    "code": "FILE_NOT_FOUND",
4    "message": "Archivo 'notas/leame.txt' no encontrado en el bucket 'bkt_abc123'."
5  }
6}

Manejo de Content-Type

Al subir un archivo, debe establecer el encabezado 

1Content-Type
al tipo MIME correcto para su archivo. Si se omite, el servicio utiliza 
1application/octet-stream
 por defecto.

Tipos MIME comunes:

Extensión Content-Type
.txt text/plain
.html text/html
.css text/css
.js application/javascript
.json application/json
.jpg / .jpeg image/jpeg
.png image/png
.gif image/gif
.pdf application/pdf
.zip application/zip
.gz application/gzip
.mp4 video/mp4

Convenciones de Rutas de Archivos

  • Las rutas de archivos distinguen entre mayúsculas y minúsculas
  • No se permiten barras inclinadas iniciales: 
    1notas/leame.txt
     (correcto), 
    1/notas/leame.txt
     (incorrecto)
  • Las rutas pueden tener hasta 1024 caracteres
  • Use barras inclinadas (
    1/
    ) como separadores de directorio
  • Los caracteres especiales en los segmentos de ruta deben codificarse como URL

Resumen de Códigos de Estado HTTP

Código Descripción
200 Éxito (GET, HEAD, PUT)
204 Sin Contenido (DELETE)
400 Solicitud Incorrecta
404 No Encontrado -- el archivo o bucket no existe
416 Rango No Satisfactorio
500 Error Interno del Servidor
