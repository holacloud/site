GET /v1/loggers/{id}/filter

使用正则表达式模式实时流式传输和过滤日志条目。

认证

需要数据凭据：

  • 1X-Instantlogs-Event-Secret: <secret>
     或 
    1Authorization: Bearer <secret>

路径参数

参数 描述 
1id
 日志记录器的唯一标识符

查询参数

参数 类型 默认值 描述 
1pattern
 string — 用于按消息内容过滤日志条目的正则表达式模式 
1follow
 bool 
1false
 如果为 
1true
，保持连接打开并流式传输新条目

请求

1curl "https://api.hola.cloud/v1/loggers/logger_xyz789/filter?pattern=error&follow=true" \
2  -H "X-Instantlogs-Event-Secret: lgs_abc123def456"

1GET /v1/loggers/logger_xyz789/filter?pattern=error&follow=true HTTP/1.1
2Host: api.hola.cloud
3X-Instantlogs-Event-Secret: lgs_abc123def456

响应

返回匹配的日志条目为 NDJSON 格式：

1{"message":"数据库连接错误","level":"error","service":"db","timestamp":"2025-06-20T14:22:31Z","id":"evt_002"}
2{"message":"超时","level":"error","service":"web","timestamp":"2025-06-20T14:22:32Z","id":"evt_003"}

当 

1follow=true
时，连接保持打开状态，新的匹配条目到达时会被流式传输。

错误代码

代码 描述
400 无效的正则表达式模式
401 缺少或无效的事件密钥
404 日志记录器未找到
