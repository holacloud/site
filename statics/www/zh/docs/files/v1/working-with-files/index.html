WorkingWithFiles

本指南涵盖 HolaCloud Files 中的文件操作：上传、下载、列出、获取元数据和删除文件。

文件路径

文件路径在 URL 中 /files/ 或 /list/ 之后指定。路径可以使用正斜杠包含目录：

1/v1/buckets/{id}/files/images/logo.png
2/v1/buckets/{id}/files/backups/db-2026-06-21.sql.gz

无需预先创建目录 -- 文件路径被视为带有斜杠的平面键。

上传文件

使用 PUT 上传文件。将 Content-Type 标头设置为适当的 MIME 类型。

上传文本文件

1curl -X PUT "https://api.hola.cloud/v1/buckets/bkt_abc123/files/notes/readme.txt" \
2  -H "Api-Key: 您的API密钥" \
3  -H "Api-Secret: 您的API密钥密码" \
4  -H "Content-Type: text/plain" \
5  --data-binary @readme.txt

上传二进制文件（图片）

1curl -X PUT "https://api.hola.cloud/v1/buckets/bkt_abc123/files/images/photo.jpg" \
2  -H "Api-Key: 您的API密钥" \
3  -H "Api-Secret: 您的API密钥密码" \
4  -H "Content-Type: image/jpeg" \
5  --data-binary @photo.jpg

上传带有元数据

您可以使用 File-Metadata 标头为文件附加自定义元数据：

1curl -X PUT "https://api.hola.cloud/v1/buckets/bkt_abc123/files/report.pdf" \
2  -H "Api-Key: 您的API密钥" \
3  -H "Api-Secret: 您的API密钥密码" \
4  -H "Content-Type: application/pdf" \
5  -H "File-Metadata: {\"author\": \"张三\", \"department\": \"工程部\"}" \
6  --data-binary @report.pdf

预期响应：

 1{
 2  "path": "report.pdf",
 3  "size": 245760,
 4  "contentType": "application/pdf",
 5  "uploadedAt": "2026-06-21T12:00:00Z",
 6  "etag": "\"abc123def456\"",
 7  "metadata": {
 8    "author": "张三",
 9    "department": "工程部"
10  }
11}

下载文件

使用 GET 通过文件路径下载：

1curl "https://api.hola.cloud/v1/buckets/bkt_abc123/files/images/photo.jpg" \
2  -H "Api-Key: 您的API密钥" \
3  -H "Api-Secret: 您的API密钥密码" \
4  -o photo.jpg

HTTP 请求：

1GET /v1/buckets/bkt_abc123/files/images/photo.jpg HTTP/1.1
2Host: api.hola.cloud
3Api-Key: 您的API密钥
4Api-Secret: 您的API密钥密码

响应包含文件内容以及适当的 Content-Type 和 Content-Length 标头。

部分下载（范围请求）

Files 支持 HTTP 范围请求以实现部分下载：

1curl "https://api.hola.cloud/v1/buckets/bkt_abc123/files/video.mp4" \
2  -H "Api-Key: 您的API密钥" \
3  -H "Api-Secret: 您的API密钥密码" \
4  -H "Range: bytes=0-1023" \
5  -o video_chunk.mp4

列出文件

使用可选的前缀过滤器列出存储桶中的文件。/list/ 后的 * 是必需的，表示所有文件。使用特定路径按前缀过滤。

列出所有文件

1curl "https://api.hola.cloud/v1/buckets/bkt_abc123/list/*" \
2  -H "Api-Key: 您的API密钥" \
3  -H "Api-Secret: 您的API密钥密码"

列出目录中的文件

列出 images/ 前缀下的文件：

1curl "https://api.hola.cloud/v1/buckets/bkt_abc123/list/images/*" \
2  -H "Api-Key: 您的API密钥" \
3  -H "Api-Secret: 您的API密钥密码"

响应：

 1{
 2  "files": [
 3    {
 4      "path": "images/photo.jpg",
 5      "size": 245760,
 6      "contentType": "image/jpeg",
 7      "modifiedAt": "2026-06-21T12:00:00Z"
 8    },
 9    {
10      "path": "images/banner.png",
11      "size": 512000,
12      "contentType": "image/png",
13      "modifiedAt": "2026-06-21T11:30:00Z"
14    }
15  ],
16  "prefix": "images/",
17  "total": 2
18}

分页列出

1curl "https://api.hola.cloud/v1/buckets/bkt_abc123/list/*?limit=10&offset=0" \
2  -H "Api-Key: 您的API密钥" \
3  -H "Api-Secret: 您的API密钥密码"

获取文件元数据

使用 HEAD 获取文件元数据，无需下载内容：

1curl -I "https://api.hola.cloud/v1/buckets/bkt_abc123/files/report.pdf" \
2  -H "Api-Key: 您的API密钥" \
3  -H "Api-Secret: 您的API密钥密码"

响应标头：

1HTTP/1.1 200 OK
2Content-Type: application/pdf
3Content-Length: 245760
4ETag: "abc123def456"
5Last-Modified: Sun, 21 Jun 2026 12:00:00 GMT
6File-Metadata: {"author": "张三", "department": "工程部"}

删除文件

按路径删除文件：

1curl -X DELETE "https://api.hola.cloud/v1/buckets/bkt_abc123/files/notes/readme.txt" \
2  -H "Api-Key: 您的API密钥" \
3  -H "Api-Secret: 您的API密钥密码"

预期响应：HTTP 204 无内容。

错误：文件未找到

1{
2  "error": {
3    "code": "FILE_NOT_FOUND",
4    "message": "在存储桶 'bkt_abc123' 中未找到文件 'notes/readme.txt'。"
5  }
6}

Content-Type 处理

上传文件时，您应将 Content-Type 标头设置为文件的正确 MIME 类型。如果省略，服务默认使用 application/octet-stream。

常见 MIME 类型：

扩展名 Content-Type
.txt text/plain
.html text/html
.css text/css
.js application/javascript
.json application/json
.jpg / .jpeg image/jpeg
.png image/png
.gif image/gif
.pdf application/pdf
.zip application/zip
.gz application/gzip
.mp4 video/mp4

文件路径约定

  • 文件路径区分大小写
  • 不允许前导斜杠：notes/readme.txt（正确），/notes/readme.txt（错误）
  • 路径最多可达 1024 个字符
  • 使用正斜杠（/）作为目录分隔符
  • 路径段中的特殊字符应进行 URL 编码

HTTP 状态码汇总

状态码 描述
200 成功 (GET, HEAD, PUT)
204 无内容 (DELETE)
400 错误请求
404 未找到 -- 文件或存储桶不存在
416 范围不满足要求
500 服务器内部错误
