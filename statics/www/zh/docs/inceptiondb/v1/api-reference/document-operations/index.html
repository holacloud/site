InceptionDB API 参考→Document Operations

Document Operations

对集合执行文档操作。具体操作由集合名称后的冒号后缀决定。

身份验证

需要 Api-Key、Api-Secret 和 X-Project 请求头。

路径参数

参数 类型 描述
id uuid 数据库的唯一标识符
col string 带操作后缀的集合名称

操作

插入 — {collection}:insert

向集合中插入一个或多个文档。

 1POST /v1/databases/a1b2c3d4-e5f6-7890-abcd-ef1234567890/collections/users:insert HTTP/1.1
 2Host: api.hola.cloud
 3Api-Key: 1abbe476-6ad6-4b97-9cca-6deb6ab2901d
 4Api-Secret: 4bda6d52-762b-4e5d-bed7-85614c13b8bf
 5X-Project: acme-webapp
 6Content-Type: application/json
 7
 8{
 9  "name": "Alice",
10  "email": "alice@example.com",
11  "role": "admin"
12}

 1curl -X POST "https://api.hola.cloud/v1/databases/a1b2c3d4-e5f6-7890-abcd-ef1234567890/collections/users:insert" \
 2  -H "Api-Key: 1abbe476-6ad6-4b97-9cca-6deb6ab2901d" \
 3  -H "Api-Secret: 4bda6d52-762b-4e5d-bed7-85614c13b8bf" \
 4  -H "X-Project: acme-webapp" \
 5  -H "Content-Type: application/json" \
 6  -d '{
 7    "name": "Alice",
 8    "email": "alice@example.com",
 9    "role": "admin"
10  }'

响应：

1{
2  "id": "d7e8f9a0-b1c2-3456-7890-123456789abc",
3  "message": "文档插入成功"
4}

查找 — {collection}:find

使用过滤条件查询文档。

 1POST /v1/databases/a1b2c3d4-e5f6-7890-abcd-ef1234567890/collections/users:find HTTP/1.1
 2Host: api.hola.cloud
 3Api-Key: 1abbe476-6ad6-4b97-9cca-6deb6ab2901d
 4Api-Secret: 4bda6d52-762b-4e5d-bed7-85614c13b8bf
 5X-Project: acme-webapp
 6Content-Type: application/json
 7
 8{
 9  "filter": {
10    "role": "admin"
11  }
12}

 1curl -X POST "https://api.hola.cloud/v1/databases/a1b2c3d4-e5f6-7890-abcd-ef1234567890/collections/users:find" \
 2  -H "Api-Key: 1abbe476-6ad6-4b97-9cca-6deb6ab2901d" \
 3  -H "Api-Secret: 4bda6d52-762b-4e5d-bed7-85614c13b8bf" \
 4  -H "X-Project: acme-webapp" \
 5  -H "Content-Type: application/json" \
 6  -d '{
 7    "filter": {
 8      "role": "admin"
 9    }
10  }'

响应：

 1{
 2  "documents": [
 3    {
 4      "id": "d7e8f9a0-b1c2-3456-7890-123456789abc",
 5      "name": "Alice",
 6      "email": "alice@example.com",
 7      "role": "admin"
 8    }
 9  ]
10}
