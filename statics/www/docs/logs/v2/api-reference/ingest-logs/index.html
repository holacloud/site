InstantLogs API Reference→Ingest Logs

Ingest Logs

Ingest log entries into a logger.

Authentication

Requires data credentials:

  • X-Instantlogs-Event-Secret: <secret> or Authorization: Bearer <secret>

Path Parameters

Parameter Description
id The unique identifier of the logger

Request Body

Single log entry:

Field Type Required Description
message string yes The log message content
level string no Log level (e.g., info, warn, error)
service string no Source service name
timestamp string no ISO 8601 timestamp (defaults to server time) 
1{
2  "message": "User login successful",
3  "level": "info",
4  "service": "web",
5  "timestamp": "2025-06-20T14:22:30Z"
6}

Request

1curl -X POST "https://api.hola.cloud/v1/loggers/logger_xyz789/ingest" \
2  -H "X-Instantlogs-Event-Secret: lgs_abc123def456" \
3  -H "Content-Type: application/json" \
4  -d '{
5    "message": "User login successful",
6    "level": "info",
7    "service": "web",
8    "timestamp": "2025-06-20T14:22:30Z"
9  }'

 1POST /v1/loggers/logger_xyz789/ingest HTTP/1.1
 2Host: api.hola.cloud
 3X-Instantlogs-Event-Secret: lgs_abc123def456
 4Content-Type: application/json
 5

11}

Response

1{
2  "ingested": 1
3}

Error Codes

Code Description
400 Missing or invalid request body
401 Missing or invalid event secret
404 Logger not found
413 Payload too large
