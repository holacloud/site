Task Lifecycle

This document details the complete lifecycle of a task: enqueuing, listing, reserving, extending leases, acknowledging, rescheduling, and real-time streaming via SSE.

Task States

A task moves through the following states:

  • pending — Waiting for its scheduled time. Not yet visible to workers.
  • available — Scheduled time has elapsed; ready for reservation.
  • reserved — Held by a worker under a lease.
  • completed — Acknowledged and removed by the worker.

If a lease expires without acknowledgement, the task returns to available.

Enqueuing a Task

Create a new task with a payload, future time (or delay), and optional labels:

 1curl -X POST "https://api.hola.cloud/schedulers/sched-a1b2c3d4-e5f6-7890-abcd-ef1234567890/tasks" \
 2  -H "X-API-Key: YOUR_API_KEY" \
 3  -H "Content-Type: application/json" \
 4  -d '{
 5    "payload": {
 6      "action": "generate_report",
 7      "report_id": "rpt-5678"
 8    },
 9    "future": "2025-06-22T00:00:00Z",
10    "labels": {
11      "team": "analytics",
12      "env": "production"
13    }
14  }'

 1POST /schedulers/sched-a1b2c3d4-e5f6-7890-abcd-ef1234567890/tasks HTTP/1.1
 2Host: api.hola.cloud
 3X-API-Key: YOUR_API_KEY
 4Content-Type: application/json
 5
 6{
 7  "payload": {
 8    "action": "generate_report",
 9    "report_id": "rpt-5678"
10  },
11  "future": "2025-06-22T00:00:00Z",
12  "labels": {
13    "team": "analytics",
14    "env": "production"
15  }
16}

Alternatively, use 

1delay
(seconds from now) instead of 
1future
:

1{
2  "payload": { "action": "send_reminder" },
3  "delay": 3600,
4  "labels": { "type": "email" }
5}

Expected response:

1{
2  "id": "task-001",
3  "scheduler_id": "sched-a1b2c3d4-e5f6-7890-abcd-ef1234567890",
4  "payload": { "action": "generate_report", "report_id": "rpt-5678" },
5  "state": "pending",
6  "available_at": "2025-06-22T00:00:00Z",
7  "labels": { "team": "analytics", "env": "production" }
8}

Reserving a Task

A worker reserves an available task by specifying a 

1worktime
— the lease duration in seconds:

1curl -X POST "https://api.hola.cloud/schedulers/sched-a1b2c3d4-e5f6-7890-abcd-ef1234567890/tasks/reserve" \
2  -H "X-API-Key: YOUR_API_KEY" \
3  -H "Content-Type: application/json" \
4  -d '{
5    "worktime": 60
6  }'

1POST /schedulers/sched-a1b2c3d4-e5f6-7890-abcd-ef1234567890/tasks/reserve HTTP/1.1
2Host: api.hola.cloud
3X-API-Key: YOUR_API_KEY
4Content-Type: application/json
5
6{
7  "worktime": 60
8}

Expected response:

1{
2  "id": "task-002",
3  "payload": { "action": "send_reminder" },
4  "lease_expires_at": "2025-06-21T12:01:00Z"
5}

If no task is available, the endpoint returns HTTP 

1204 No Content
.

Extending a Lease

If the worker needs more time, extend the lease before it expires:

1curl -X POST "https://api.hola.cloud/schedulers/sched-a1b2c3d4-e5f6-7890-abcd-ef1234567890/tasks/task-002/extend" \
2  -H "X-API-Key: YOUR_API_KEY" \
3  -H "Content-Type: application/json" \
4  -d '{
5    "worktime": 30
6  }'

1POST /schedulers/sched-a1b2c3d4-e5f6-7890-abcd-ef1234567890/tasks/task-002/extend HTTP/1.1
2Host: api.hola.cloud
3X-API-Key: YOUR_API_KEY
4Content-Type: application/json
5
6{
7  "worktime": 30
8}

Expected response:

1{
2  "id": "task-002",
3  "lease_expires_at": "2025-06-21T12:01:30Z"
4}

Acknowledging (Deleting) a Task

After successful processing, acknowledge the task to remove it:

1curl -X DELETE "https://api.hola.cloud/schedulers/sched-a1b2c3d4-e5f6-7890-abcd-ef1234567890/tasks/task-002" \
2  -H "X-API-Key: YOUR_API_KEY"

1DELETE /schedulers/sched-a1b2c3d4-e5f6-7890-abcd-ef1234567890/tasks/task-002 HTTP/1.1
2Host: api.hola.cloud
3X-API-Key: YOUR_API_KEY

Expected response: HTTP 

1204 No Content
.

Rescheduling a Task

If processing fails, reschedule the task with a new delay or future time:

1curl -X POST "https://api.hola.cloud/schedulers/sched-a1b2c3d4-e5f6-7890-abcd-ef1234567890/tasks/task-002/reschedule" \
2  -H "X-API-Key: YOUR_API_KEY" \
3  -H "Content-Type: application/json" \
4  -d '{
5    "delay": 300
6  }'

1POST /schedulers/sched-a1b2c3d4-e5f6-7890-abcd-ef1234567890/tasks/task-002/reschedule HTTP/1.1
2Host: api.hola.cloud
3X-API-Key: YOUR_API_KEY
4Content-Type: application/json
5
6{
7  "delay": 300
8}

Expected response:

1{
2  "id": "task-002",
3  "state": "pending",
4  "available_at": "2025-06-21T12:06:00Z"
5}

This is ideal for implementing retry logic with exponential backoff.

SSE Stream for Real-Time Updates

Subscribe to real-time task events via Server-Sent Events:

1curl "https://api.hola.cloud/schedulers/sched-a1b2c3d4-e5f6-7890-abcd-ef1234567890/tasks/stream"

The stream emits events in SSE format:

1event: task_available
2data: {"id":"task-003","payload":{"action":"process_image"},"labels":{}}
3
4event: task_completed
5data: {"id":"task-003"}
6
7event: task_expired
8data: {"id":"task-003"}
Event Description 
1task_available
 A task has become available for reservation 
1task_completed
 A task was acknowledged by a worker 
1task_expired
 A lease expired and the task is available again

SSE is useful for building responsive worker pools that react to tasks immediately without polling.
